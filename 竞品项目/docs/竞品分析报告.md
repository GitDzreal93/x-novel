# 🔍 火果小说 AI 生成器 - 竞品分析报告

## 项目概述

**项目名称**: 火果小说 AI 生成器 (huobao-novel)
**体验地址**: https://marketing.chatfire.site/huobao-novel/
**技术栈**: Vue 3 + Vite + Naive UI + Tailwind CSS + Pinia
**定位**: 基于雪花写作法的 AI 小说创作工具，面向 C 端用户的智能写作辅助平台

---

## 📊 产品定位与目标用户

### 核心定位
- **To C 产品**: 面向个人创作者、网络小说作者、小说爱好者的 AI 辅助创作工具
- **创作方法**: 基于**雪花写作法**（Snowflake Method）的系统化小说生成流程
- **核心价值**: 降低创作门槛，提升创作效率，保持创作连贯性

### 目标用户画像
1. **新手作者**: 有创意但缺乏写作经验的创作者
2. **网文作者**: 需要快速产出和保持更新频率的网络小说作者
3. **小说爱好者**: 希望尝试创作但不知从何下手的爱好者
4. **内容创作者**: 需要多平台内容产出的自媒体创作者

### 2. 小说架构生成（核心功能）

#### 雪花写作法 5 步骤

**步骤 1: 核心种子 (Core Seed)**
- 单句概括故事本质
- 根据不同类型生成不同公式
- 字数要求：25-100 字
- 示例公式：
  - 悬疑类："当[主角]遭遇[核心事件]，必须[关键行动]，否则[灾难后果]..."
  - 言情类："当[主角]在[背景环境]中遇见[另一角色]..."
  - 玄幻类："当[主角]获得[机缘]，踏上[修炼之路]..."

**步骤 2: 角色动力学 (Character Dynamics)**
- 3-6 个核心角色设计
- 每个角色包含：
  - 特征：背景、外貌、性别、年龄、职业
  - 核心驱动力三角：表面追求 → 深层渴望 → 灵魂需求
  - 角色弧线：初始状态 → 触发事件 → 内心转变 → 成长节点 → 最终状态
  - 角色关系网：与其他角色的关系、互动模式、情感连接点、需要跨越的障碍

**步骤 3: 角色状态 (Character State)**
- 结构化角色状态文档
- 包含：
  - 物品（装备、道具）
  - 能力（技能、特长）
  - 状态（身体状态、心理状态）
  - 主要角色间关系网
  - 触发或加深的事件

**步骤 4: 世界观构建 (World Building)**
- 三维交织法：
  - 物理维度：空间结构、时间背景、规则体系
  - 社会维度：社会结构、文化氛围、生活方式
  - 情感维度：核心意象、环境氛围与情感的呼应
- 每维度至少 3 个动态元素

**步骤 5: 情节架构 (Plot Architecture)**
- 三幕式结构：
  - 第一幕（开端）：日常状态展示 → 契机事件 → 初步反应
  - 第二幕（发展）：剧情深入 → 挑战与成长 → 重要转折
  - 第三幕（高潮与结局）：核心冲突爆发 → 角色抉择 → 情感/事件收尾

#### UI 交互设计
- 折叠面板展示各模块
- 渐变图标区分不同类型
- 可编辑文本框（支持手动修改）
- 状态标签显示生成状态

### 3. 章节大纲生成

#### 功能特点
- **悬念节奏曲线设计**
  - 每 3-5 章构成一个情节单元
  - 单元之间安排情感节奏（张弛有度）
  - 关键转折章预留铺垫

- **每章信息包含**
  - 章节定位（角色/事件/主题等）
  - 核心作用（推进/转折/发展/升温等）
  - 情感强度（平缓/渐进/高潮等）
  - 伏笔操作（埋设/强化/回收）
  - 情节张力（★☆☆☆☆ 到 ★★★★★）
  - 本章简述（一句话概括）

- **智能分块生成**
  - 根据模型 maxTokens 自动计算分块大小
  - 支持断点续传（已有章节保留）
  - 限制已有大纲到最近 100 章（避免超出上下文）

### 4. 章节写作面板

#### 功能模块

**章节列表侧边栏**
- 显示所有章节大纲
- 章节完成状态标记（绿色对勾）
- 当前章节高亮
- 点击切换章节

**章节信息头部**
- 章节标题
- 状态标签（已保存/未保存）
- 元信息标签（定位、作用、悬念等级）
- 章节简述

**前文摘要区域**
- 仅第 2 章及以后显示
- 琥珀色背景高亮
- 可滚动查看

**写作操作按钮**
- **生成草稿**: AI 生成章节正文
- **扩写**: 对现有内容进行扩写
- **快速保存**: 仅保存内容，不更新上下文
- **保存并定稿**: 保存内容 + 更新摘要 + 更新角色状态

**编辑器**
- 自动调整高度的文本框
- 实时字数统计
- 支持手动编辑

**章节关系图谱（内嵌）**
- 显示本章涉及的角色和关系
- 可视化交互图

#### 上下文连贯性机制
- **前文摘要**: 每章定稿时自动更新，累积前面所有章节的关键信息
- **角色状态**: 每章定稿时更新，记录角色物品、能力、心理状态、关系变化
- **前章结尾段**: 生成时传入前一章最后 800 字作为衔接

### 5. 灵感罗盘（关系图谱）

#### 核心功能

**基线图谱生成**
- 从架构数据中提取实体关系
- 实体类型：character（角色）、faction（阵营）、location（地点）、item（物品）
- 关系类型：hostile（敌对）、romantic（情感）、alliance（同盟）、neutral（中立）、family（亲属）、mentor（师徒）

**章节快照**
- 每章完成后自动生成增量变化
- 记录节点的增删改
- 记录关系的增删改
- 支持按章节回溯查看

**交互功能**
- **节点点击**: 显示角色详情侧边栏
- **关系点击**: 显示关系详情弹窗
- **双击节点**: 聚焦模式（仅显示该节点及其关联）
- **时间轴**: 拖动查看不同章节的关系状态

**导出功能**
- 导出 PNG 图片
- 导出 HTML（可交互）

#### 数据结构
```javascript
graphData: {
  version: 1,
  generatedAt: '时间戳',
  snapshots: {
    '0': { nodes: [...], edges: [...] },
    '5': { nodes: [...], edges: [...] }

  graphGenerated: true

```

## 💡 AI 提示词工程分析

### 架构类提示词特点

1. **类型感知**: 所有提示词都包含 `根据【${genre}】类型特点` 的指导
2. **模板化**: 为不同类型提供不同的公式模板
3. **渐进式**: 从抽象到具体，从核心到细节
4. **约束明确**: 字数要求、格式要求、内容要求都有明确说明

### 章节类提示词特点

1. **上下文丰富**: 传入前文摘要、前章结尾、角色状态、下一章目录
2. **场景设计**: 要求 AI 设计 2+ 个场景（对话、动作、心理、环境）
3. **格式约束**: 不使用 markdown，不分小标题
4. **连贯性保证**: 强调与前后章节的衔接

### Compass 提示词特点

1. **JSON 严格输出**: 多次强调仅输出 JSON，不要代码块标记
2. **增量分析**: 分析章节内容的变化（delta）
3. **逻辑审计**: 检测 5 种逻辑问题（dead_reappear、missing_setup 等）
4. **实体分类**: 明确节点类型前缀规则（char_、fac_、loc_、item_）

## 📦 技术架构分析

#### 状态管理（Pinia）
- **novel.js**: 小说项目状态
  - projects: 项目列表

  - apiConfig: API 配置

- `/`: 首页（项目列表）
- `/project/:id`: 项目详情页
- 使用 5 个子标签页（架构、大纲、写作、罗盘、导出）

#### 数据持久化
- 所有数据存储在 localStorage
- 键名：`novel_projects`、`api_config`、`stage_models`、`theme`
- 项目更新时自动同步到 localStorage

### API 层设计

#### llm.js - LLM 交互
```javascript
chatCompletion(config, prompt, onStream)
- 支持流式和非流式
- OpenAI 兼容接口
- 超时控制
- cleanResponse: 清理 markdown 格式
```

#### generator.js - 小说生成逻辑
```javascript
generateArchitecture() // 5 步架构生成
```

#### compass-generator.js - 关系图谱
```javascript

generateChapterSnapshots() // 批量生成章节快照
```

## 🚀 优化建议（人有我优）

基于竞品分析和真实小说作者的创作流程，以下是我们可以超越的核心方向：

---

### 📖 第一阶段：灵感涌现期

**目标**: 帮助作者从 0 到 1，快速找到创作方向

#### 1. 智能问答激发灵感
```
交互方式：ChatGPT 式对话
├── 创意启发
│   ├── "我想写一个XX类型的故事，有什么新颖的切入点？"
│   ├── "给我 10 个可能的情节核心冲突"
│   └── "帮我设定一个令人印象深刻的反派角色"
│
├── 世界观构建对话
│   ├── "我的设定是XXX，有哪些逻辑漏洞？"
│   ├── "如何让这个世界更有沉浸感？"
│   └── "帮我想 5 个这个世界独有的特色"
│
└── 角色设定对话
    ├── "这个角色的性格还可以更立体一些"
    ├── "帮我想一下他的成长轨迹"
    └── "他和主角之间可以有什么复杂的关系？"
```

#### 2. 创意拼装器
- **创意卡片**: 将常见元素（角色类型、情节模式、世界观设定）做成卡片
- **随机组合**: 一键随机生成 3-5 个创意组合供参考
- **收藏夹**: 保存喜欢的创意片段

#### 3. 热门题材分析
- 实时抓取主流平台热门题材、标签
- 分析市场趋势（什么类型更受欢迎）
- 提供创作建议

---

### ✍️ 第二阶段：写作辅助期

**目标**: 提升写作效率，保证内容质量

#### 1. 智能写作助手（右侧边栏）

## 📈 技术实现要点

### 关键技术点

1. **流式响应处理**: 使用 Fetch API + ReadableStream
2. **图谱可视化**: 可能使用 ECharts 或 D3.js
3. **状态持久化**: localStorage + Pinia
4. **主题切换**: CSS 类切换 + 系统偏好检测
5. **提示词模板**: 模板字符串 + 参数插值

### 性能优化

1. **分块生成**: 避免超出模型上下文限制
2. **增量更新**: 仅更新变化的数据
3. **懒加载**: 章节内容按需加载
4. **防抖节流**: 输入和搜索优化

## 🎯 总结

火果小说 AI 生成器是一个设计精良、功能完善的 AI 创作工具。其核心优势在于：

1. **系统化的创作方法**: 基于雪花写作法的完整流程
2. **类型感知的提示词**: 根据不同小说类型定制生成策略
3. **上下文连贯性保证**: 通过摘要、角色状态、前章结尾等多重机制确保连贯性
4. **关系图谱可视化**: 创新的灵感罗盘功能，直观展示角色关系演变
5. **良好的 UI/UX**: 渐变色设计、深色模式、流畅动画

我们的复刻目标是：
- **功能 1:1 对等**: 确保所有核心功能都有
- **体验人有我优**: 在交互、动画、视觉、性能上超越
